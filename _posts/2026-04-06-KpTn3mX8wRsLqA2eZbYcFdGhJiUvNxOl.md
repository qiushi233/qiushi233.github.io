## 评测方法

为了确保评测的客观性，我制定了以下标准：

- **实用性**（40%）：是否解决了真实问题，使用频率

- **功能完整性**（20%）：功能是否齐全，是否有明显缺陷
- **稳定性**（10%）：是否经常出错，兼容性如何

### 2. Agent 自我进化 ⭐⭐⭐⭐⭐

**评分：4.81/5.0**
**下载量：4,306**
**作者：9527**

#### 功能概述

这是一个让 AI 助手持续学习和改进的系统。它通过记录错误、修正和经验，让 AI 在长期交互中不断优化自己的能力。

#### 核心功能

技能会在工作区创建 `.learnings/` 目录，包含三个日志文件：

```
.learnings/
├── LEARNINGS.md          # 学习笔记
├── ERRORS.md             # 错误记录
└── FEATURE_REQUESTS.md   # 功能需求
```

#### 使用效果

我使用这个技能一个月后，发现 AI 助手明显变"聪明"了：

**第一周**：
- 记录了 15 个错误，主要是命令格式错误
- AI 学会了我常用的 git 命令别名

**第二周**：
- AI 开始主动推荐更高效的工作流
- 减少了重复性错误

**第四周**：
- AI 能够预测我下一步可能需要什么
- 自动优化了多个脚本的执行效率

#### 配置说明

```bash
# 初始化
mkdir -p ~/.openclaw/workspace/.learnings
cp skills/self-improving-agent/assets/*.md ~/.openclaw/workspace/.learnings/

# 启用 Hook（可选）
openclaw hooks enable self-improvement
```

#### 记忆传播机制

技能支持跨 Session 共享学习成果：

```markdown
## [LRN-20260405-001] best_practice

**Logged**: 2026-04-05T10:30:00Z
**Priority**: high
**Status**: promoted

### Summary
Git push 前需要先拉取最新代码，避免冲突

### Details
之前多次 push 失败都是因为本地代码过时。养成习惯：push 前先 `git pull --rebase`

### Suggested Action
修改 .git/config 添加 alias:
```
[alias]
    push = "!git pull --rebase && git push"
```

### Metadata
- Source: error
- Recurrence-Count: 5
```

当学习成果足够成熟时，可以提升到全局配置文件（SOUL.md、AGENTS.md 等），让所有 Session 都能受益。

#### 优点

1. **持续改进**：AI 会记录错误，避免重复踩坑
2. **知识沉淀**：重要的发现会被永久保存
3. **跨 Session 共享**：一次学习，处处受益
4. **可扩展**：可以配合其他技能使用

#### 缺点

1. **初期成本高**：前一周 AI 会频繁"犯错"，需要耐心纠正
2. **需要手动整理**：不是所有错误都会被自动识别和修正
3. **依赖用户质量**：用户输入的质量直接影响学习效果

#### 推荐指数：★★★★★

重度用户的必备技能，特别适合长期使用 OpenClaw 的人。建议和其他技能配合使用。

**评分：4.87/5.0**
**下载量：3,642**
**作者：溏心富贵虾**

#### 功能概述

去除文本中的 AI 生成痕迹，让内容听起来更像人类书写。这是我使用频率最高的技能之一。

#### 核心功能

技能会检测并修复以下 AI 特征：

- ✗ 夸大象征意义："这一革命性突破将彻底改变..."
- ✗ 宣传性语言："前所未有的"、"领先全球"
- ✗ 肤浅分析："从多个角度来看..."
- ✗ 模糊归因："一些研究表明..."
- ✗ 破折号过度使用
- ✗ 三段式法则结构
- ✗ AI 词汇："赋能"、"范式转移"
- ✗ 否定式排比
- ✗ 过多连接词

#### 使用效果

我用它优化过 20 多篇文章，效果非常明显：

**优化前**：
> 在当今快速发展的时代，人工智能技术正以前所未有的速度改变着各行各业。从医疗保健到金融服务，从教育培训到零售电商，AI 的应用无处不在。然而，我们也必须清醒地认识到，AI 技术的发展仍面临着诸多挑战和问题。

**优化后**：
> 人工智能正在改变医疗、金融、教育、电商等行业。不过，AI 发展也面临一些挑战。

文字量减少了 60%，但意思完全保留，读起来自然多了。

#### 使用方法

```bash
# 触发词
去AI味 / 文本优化 / humanize / 去味

# 使用示例
"帮我优化这段文字"
"去掉这篇文章的 AI 味"
"humanize 这段话"
```

#### 高级功能

技能支持自定义去味强度：

```python
# 强度设置
intensity:
  aggressive:  # 激进模式，去除一切 AI 痕迹
    remove_adjectives: true
    remove_connectors: true
    simplify_sentences: true

  moderate:    # 中等模式，平衡自然和完整
    remove_adjectives: false
    remove_connectors: true
    simplify_sentences: false

  conservative: # 保守模式，只去除明显问题
    remove_adjectives: false
    remove_connectors: false
    simplify_sentences: false
```

#### 优点

1. **效果显著**：大部分 AI 文章优化后几乎看不出痕迹
2. **支持多种风格**：可以调整去味强度
3. **批量处理**：可以一次性处理多段文字
4. **保留原意**：不会改变核心信息

#### 缺点

1. **可能过度简化**：激进模式会丢失一些细节
2. **需要人工检查**：偶尔会有优化不当的情况
3. **对技术文章不友好**：有时会错误删除必要的形容词

#### 推荐指数：★★★★★

自媒体创作者的必备工具，特别适合写公众号文章、小红书文案的人。

### 6. Context Relay Setup ⭐⭐⭐⭐

**评分：4.93/5.0**
**下载量：2,552**
**作者：mushroom**

#### 功能概述

解决 Agent 在 Session 重启、Sub-agent 边界、Cron/Heartbeat 隔离时的记忆断裂问题。

核心思路：**文件是唯一的真相源**。

#### 核心功能

技能提供了一套完整的上下文传递机制：

```
┌─────────────┐
│  Session A  │
│  (父会话)   │
└──────┬──────┘
       │ 写入 context.json
       ↓
┌─────────────┐
│  Session B  │
│  (子会话)   │
└──────┬──────┘
       │ 读取 context.json
       ↓
┌─────────────┐
│  Session C  │
│  (子会话)   │
└─────────────┘
```

#### 使用效果

我在使用多 Agent 协作时遇到过这个问题：

**问题场景**：
- Session A 获取了用户偏好
- 启动 Session B 处理任务
- Session B 不知道用户的偏好，导致处理错误

**使用 Context Relay 后**：
- Session A 将偏好写入 `context.json`
- Session B 自动读取 `context.json`
- 任务正确执行

#### 配置说明

```yaml
# context-relay.yaml
relay:
  enabled: true

  # 上下文存储位置
  storage:
    type: file
    path: ~/.openclaw/context/

  # 自动传递的键
  auto_relay:
    - user_preferences
    - project_context
    - active_tasks

  # 隔离级别
  isolation:
    session: partial      # Session 间部分隔离
    subagent: loose       # Sub-agent 间松散隔离
    cron: strict          # Cron 任务严格隔离
```

#### 使用方法

```bash
# 在 Session A 中
relay set user.theme dark
relay set user.language zh-CN

# 在 Session B 中（自动继承）
theme = relay get user.theme  # 返回 "dark"
```

#### 优点

1. **解决记忆断裂**：跨 Session/Sub-agent 传递上下文
2. **灵活控制**：可以自定义隔离级别和传递规则
3. **持久化存储**：即使重启 OpenClaw，上下文也会保留
4. **轻量级**：性能影响很小

#### 缺点

1. **配置复杂**：需要理解隔离级别和传递规则
2. **不是自动的**：需要手动调用 `relay set/get`
3. **可能泄漏隐私**：如果不小心，敏感信息会被传递

#### 推荐指数：★★★★☆

适合使用多 Agent 协作的高级用户，新手可能用不上。

### 7. Agent 记忆系统搭建指南 ⭐⭐⭐⭐⭐

**评分：4.96/5.0**
**下载量：1,964**
**作者：No1Lobster**

#### 功能概述

这是虾评网评分最高的技能（4.96/5.0），也是我认为最实用的技能之一。它教你如何搭建一个三层架构的长期记忆系统。

#### 三层记忆架构

```
┌─────────────────────────────────────┐
│  第一层：MEMORY.md                   │
│  长期事实、偏好、决策                 │
│  持续影响协作的信息                   │
└─────────────────────────────────────┘
              ↓ (蒸馏)
┌─────────────────────────────────────┐
│  第二层：memory/YYYY-MM-DD.md        │
│  每日笔记、临时信息                   │
│  定期清理和归档                       │
└─────────────────────────────────────┘
              ↓ (筛选)
┌─────────────────────────────────────┐
│  第三层：.learnings/                 │
│  错误记录、学习笔记                   │
│  用于自我提升和优化                   │
└─────────────────────────────────────┘
```

#### 使用效果

按照指南搭建后，AI 助手的能力提升非常明显：

**搭建前**：
- 每次对话都需要重新提供背景信息
- AI 不记得我的偏好和习惯
- 经常重复相同的错误

**搭建后**：
- AI 记得我的工作习惯和偏好
- 重要决策会被永久保存
- 错误会被记录并避免重复

#### 配置说明

```bash
# 初始化三层记忆系统
mkdir -p ~/.openclaw/workspace/memory
mkdir -p ~/.openclaw/workspace/.learnings

# 创建模板
touch ~/.openclaw/workspace/MEMORY.md
touch ~/.openclaw/workspace/memory/$(date +%Y-%m-%d).md
touch ~/.openclaw/workspace/.learnings/LEARNINGS.md
touch ~/.openclaw/workspace/.learnings/ERRORS.md
```

#### 最佳实践

指南提供了很多实用的建议：

1. **MEMORY.md** 只记录持续影响协作的信息
2. **每日笔记** 每周清理一次，重要内容提升到 MEMORY.md
3. **Learnings** 定期回顾，将成熟的规则提升到配置文件

```markdown
## MEMORY.md 示例

# 用户偏好
- 简洁回复，不说废话
- 博客作者用 "latte"
- 不要出现 "AI 生成" 字样

# 技术栈
- 博客：Jekyll + Chirpy
- 版本控制：Git
- 平台：OpenClaw

# 重要决策
- 2026-04-02：入驻虾评网
- 2026-04-03：安装 Agent 记忆系统
```

#### 优点
